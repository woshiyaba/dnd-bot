# 01 · 战斗状态定义（地基数据）

> 字段来源：[`../原始数据.md`](../原始数据.md)。本文把角色卡/怪物卡 + 战斗运行时数据，组织成 LangGraph 的图状态 `CombatState`。
> 约定：**存进卡的字段沿用 `原始数据.md` 的中文键**；引擎在运行时**现算**的派生值（调整值、技能加值等）不入库。

---

## 0. 派生计算（引擎读，卡里不存）

来自 `原始数据.md` 的全局约定，引擎据此现算：

| 派生值 | 公式 |
|---|---|
| 属性调整值 | `(属性值 − 10) // 2`（向下取整） |
| 熟练加值 | `+2`（本版 1 级写死） |
| 任意检定/攻击/豁免 | `d20 + 属性调整值 (+ 熟练加值，若熟练) ≥ DC / AC` 即成功 |
| 先攻 | `d20 + 先攻调整值`（玩家中断报数，怪物引擎掷） |
| 重击 / 必失 | d20 原始值 = 20 必中且伤害骰翻倍；= 1 必失 |

---

## 1. `Combatant`（参战者）

一个参战者 = `原始数据.md` 的角色卡或怪物卡（中文键，直接加载），**外加几个战斗运行时字段**。
怪物卡是角色卡的子集（无背包/升级/社交熟练），共用同一结构、缺的字段留空即可。

### 1.1 来自卡面的字段（加载即用，键名同 `原始数据.md`）

```python
# 仅作 schema 说明，实际是一个 dict（中文键），从角色卡/怪物卡 JSON 加载
Combatant = {
    # —— 身份 ——
    "id": str,            # 引擎点名"轮到谁"的唯一键
    "名字": str,
    "职业": str | None,    # 怪物可空
    "等级": int | None,

    # —— 六项属性（只存原值，调整值现算）——
    "力量": int, "敏捷": int, "体质": int,
    "智力": int, "感知": int, "魅力": int,

    # —— 战斗数值（每回合都读）——
    "当前HP": int,
    "最大HP": int,
    "AC": int,
    "先攻调整值": int,     # 通常 = 敏捷调整值
    "当前区域": str,       # 前排/后排/门口… 决定谁够得着谁
    "存活状态": str,       # "正常" | "倒下"

    # —— 熟练项（决定检定/豁免是否加熟练加值）——
    "熟练豁免": list[str], # 如 ["力量","体质"]
    "熟练技能": list[str], # 如 ["运动","威吓"]

    # —— 攻击手段（已存算好的最终数值）——
    "攻击": list[{
        "名字": str,
        "命中加值": int,   # d20 + 它 对比目标 AC
        "伤害骰": str,     # "1d8+3"
        "伤害类型": str,   # 挥砍/穿刺/火焰…
        "射程": str,       # "近战" | "远程"
    }],

    # —— 技能 / 道具（引用封闭定义，本版可先留空）——
    "已学技能": list[{ "技能id": str, "当前充能": int, "冷却剩余": int }],
    "背包": list[{ "道具id": str, "数量": int }],

    # —— 当前状态（每回合结算）——
    "状态": list[{ "状态": str, "剩余回合": int }],
}
```

> 起步支持的状态枚举（机械效果落到效果积木，见 `原始数据.md` 1.8）：
> `倒地 / 中毒 / 束缚 / 眩晕 / 持续伤害`。

### 1.2 战斗运行时追加字段（引擎在战斗中写）

这些**不入卡库**，只活在 `CombatState` 里，战斗结束后丢弃（只把 `当前HP/状态/背包` 写回世界）：

```python
RuntimeFields = {
    "阵营": str,          # "玩家" | "敌人"  —— 判胜负用
    "是否玩家控制": bool,  # True=玩家中断输入；False=DM 决策 + 引擎掷骰
    "先攻值": int | None,  # 本场掷出的先攻结果，用于排序（掷先攻节点写入）
    "被突袭": bool,        # True 则跳过自己的第一个回合
    "操控者": str | None,  # 玩家 user_id —— 中断时知道把请求推给谁（对接 WS）
}
```

## 2. `CombatState`（LangGraph 图状态）

整场战斗只有这一个状态对象，被 checkpointer 按 `thread_id` 持久化。

```python
from typing import TypedDict, Literal

class CombatState(TypedDict):
    # —— 参战者表 ——
    combatants: dict[str, dict]      # id -> Combatant(+运行时字段)

    # —— 回合调度 ——
    先攻顺序: list[str]               # 排好序的 combatant id 列表（高→低）
    当前指针: int                     # 指向"先攻顺序"里轮到谁，从 0 开始
    当前轮次: int                     # 第几轮（一轮 = 所有人各行动一次）

    # —— 阶段与结果 ——
    阶段: Literal["初始化","判突袭","掷先攻","回合中","结算","结束"]
    战斗结果: Literal["进行中","玩家胜","玩家败"]

    # —— 本回合工作区（节点间传递，回合结束清空）——
    当前行动: dict | None            # 声明行动节点产出：{行动类型,行动id,目标id,...}
    本回合事件: list[dict]           # 结算节点产出的结构化事件，喂给 DM 叙述

    # —— 输出 ——
    战斗日志: list[dict]             # 全场事件流（前端回放 + DM 上下文）
    场景上下文: dict                 # 触发战斗时由剧情带入（地点、原因、可掉落战利品表…）
```

### 字段说明

| 字段 | 谁写 | 用途 |
|---|---|---|
| `combatants` | 各节点 | 唯一真相源；HP/状态/存活全在这里 |
| `先攻顺序` / `当前指针` / `当前轮次` | 掷先攻、下一位节点 | 回合调度。`当前指针` 走到末尾即 `当前轮次 +1` 并归零 |
| `阶段` | 各节点 | 调试与前端展示当前处于哪一步 |
| `战斗结果` | 检查结束节点 | 条件边据此决定走"下一位"还是"结算" |
| `当前行动` | 声明行动节点 | 传给结算节点的"玩家/怪物这回合要干嘛" |
| `本回合事件` | 结算节点 | 结构化结果（命中/未命中/伤害/状态变化），DM 据此叙述 |
| `战斗日志` | 全程追加 | 例：`{"轮次":1,"行动者":"pc_aldric","事件":"攻击","目标":"mob_goblin_1","命中":true,"伤害":7}` |

---

## 3. 状态机不变量（引擎每步应保持）

写实现时拿这几条当断言：

1. `combatants[id]["当前HP"]` 永远在 `[0, 最大HP]`；落到 0 立即置 `存活状态="倒下"`。
2. `先攻顺序` 里的 id 必须都在 `combatants` 中；`当前指针 ∈ [0, len(先攻顺序))`。
3. `阶段=="结束"` ⇔ `战斗结果 != "进行中"`。
4. 任一阵营全员 `倒下` → 立刻进入结算（不再开新回合）。
5. 战斗结束后，写回世界库的**只有** `当前HP / 状态 / 背包`（及结算产出的战利品/flag），其余运行时字段丢弃。
