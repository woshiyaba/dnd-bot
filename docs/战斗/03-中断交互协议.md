# 03 · 中断交互协议（骰子交给玩家）

> 玩家要掷的骰子，全部用 LangGraph `interrupt()` 收集。本文定义：**在哪些点中断、请求长什么样、玩家怎么回、怎么和现有 WebSocket 对接**。
> 节点位置见 [`02-LangGraph流程设计.md`](./02-LangGraph流程设计.md)。

---

## 1. 中断点清单

| 中断类型（interrupt_type） | 触发节点 | 向谁要 | 要什么 |
|---|---|---|---|
| `roll_initiative`（掷先攻） | `roll_initiative` | 每个玩家 | 一个 d20 原始值 |
| `declare_action`（声明行动） | `declare_action` | 当前回合玩家 | 选哪个动作 + 目标 |
| `attack_roll`（攻击检定） | `resolve_action` | 攻击方玩家 | 攻击 d20 原始值 |
| `damage_roll`（伤害掷骰） | `resolve_action` | 攻击方玩家 | 伤害骰结果（命中后；重击则翻倍骰数） |
| `saving_throw`（豁免检定） | `resolve_action` | 被作用方玩家 | 豁免 d20 原始值 |
| `ability_check`（属性检定） | `resolve_action` | 行动方玩家 | 创意动作的检定 d20 |

> 怪物/环境的同类骰子**不中断**，由引擎自动掷。

---

## 2. 中断请求格式（图 → 前端）

> 字段键已全部英文化（与 `src/combat`、`src/model` 代码一致）。中断类型 / 行动类型 / 属性等取值为枚举值（小写英文，见 `src/model/enums.py`）。

节点里 `value = interrupt(请求)`，`请求` 统一为：

```python
{
    "interrupt_type": str,        # 见上表（枚举值：roll_initiative / declare_action / ...）
    "directed_to": {              # 面向：推给谁
        "combatant_id": str,
        "user_id": str,           # = Combatant.controller，前端据此推给正确的人
    },
    "prompt": str,                # 提示：给玩家看的话，如 "轮到奥德里克，掷先攻：d20 + 1"
    "required_dice": str | None,  # 需要骰子：骰子表达式，如 "d20" / "1d8+3"；纯选择类为 null
    "bonus": int,                 # 加值：引擎会替你加的固定值（前端可显示 "+1"）
    "options": dict | None,       # 选项：仅 declare_action 用，可选动作与合法目标
    "expected_return": dict,      # 期望返回：该类型恢复值的 schema（见第 3 节）
    # "extra": dict               # 附带：可选，如攻击检定一并带回 {"damage_dice": "1d8+3"}
}
```

### 2.1 `declare_action`（声明行动）的 `options` 结构

```python
"options": {
    "attack": [
        {"attack_name":"长剑","range":"melee",
         "targets":[{"id":"mob_goblin_1","name":"哥布林","zone":"前排"}]},
        {"attack_name":"手弩","range":"ranged","targets":[...]},   # 已按射程/区域过滤掉够不着的
    ],
    "skill": [{"skill_id":"skill_second_wind","charges_left":1,"cooldown_left":0}],
    "item": [{"item_id":"item_healing_potion","quantity":2}],
    "move": [{"target_zone":"后排"},{"target_zone":"门口"}],   # 可选
    "improvise": true,   # 允许自由文本
}
```

## 3. 恢复值格式（前端 → 图）

前端收集玩家输入后，用 `graph.invoke(Command(resume=恢复值), config={thread_id})` 恢复。
各中断类型的 `恢复值`：

```python
# roll_initiative / attack_roll / saving_throw / ability_check
{ "d20": 17 }                    # 玩家报的 d20 原始值；引擎自己加 "bonus"

# damage_roll
{ "result": 7 }                  # 玩家把整条伤害骰算好的总和报上来

# declare_action（攻击示例）
{ "action_type":"attack", "attack_name":"长剑", "target_id":"mob_goblin_1" }
# declare_action（技能/道具/创意/移动）
{ "action_type":"skill", "skill_id":"skill_second_wind", "target_id":"self" }
{ "action_type":"item", "item_id":"item_healing_potion", "target_id":"self" }
{ "action_type":"improvise", "description":"我踹翻火盆把火星溅向哥布林" }
{ "action_type":"move", "target_zone":"后排" }   # 移动后通常还需再发一次 declare_action 收主要动作
```

> **信任边界**：玩家只报 `d20` 原始值，**加值与命中判定一律引擎算**——前端别替玩家算总和，避免作弊和口径不一致。可做范围校验（1–20）。

## 4. 减少往返：攻击"一次中断收齐"（推荐）

一次玩家攻击天然有 2 次掷骰（命中、伤害）。逐个中断 = 2 个往返，多人会很慢。优化：

- `attack_roll` 的请求里把伤害骰也带上（放进 `extra`），让玩家**同时报攻击和伤害**：
  ```python
  请求 = { "interrupt_type":"attack_roll", ..., "required_dice":"d20", "extra":{"damage_dice":"1d8+3"} }
  恢复值 = { "d20":17, "damage_result":7 }   # 未命中则引擎忽略 damage_result
  ```
- 引擎用 `d20` 判命中，命中才采用 `damage_result`，未命中丢弃。
- 重击（d20=20）需要翻倍骰数时，引擎检测到后**补一个** `damage_roll` 中断要额外的骰子（重击较少，单独要可接受）。

> 这是 UX 取舍，不影响规则正确性。先做"逐个中断"最简单，跑通后再合并。

## 5. 与现有 WebSocket / FastAPI 对接

现项目（见 `app.py` / `graph.py` / `ws_manager.py`）已有：`/ws/{user_id}` 每用户连接、`manager` 按 `user_id` 推送、`astream_agent_collect` 推流 token。战斗中断接进去：

```
玩家A 连 /ws/A、玩家B 连 /ws/B …（同一房间）
            │
POST /combat/{房间id}/action  → graph.invoke(Command(resume=恢复值),
            │                                 config={"thread_id": f"combat:{房间id}"})
            ▼
图推进到下一个 interrupt() → invoke 返回带 __interrupt__ 的结果
            │
读 中断请求.directed_to.user_id → manager 把"该谁掷什么"推给那个玩家的 ws
            │
该玩家前端弹"掷 d20"→ 回报 → 再次 POST /combat/{房间id}/action 恢复
```

要点：
1. **`thread_id` 按房间唯一**（`combat:{房间id}`），替换现有硬编码 `thread_123`；checkpointer 换持久化版（SqliteSaver/自建 MySQL saver），否则重启丢档。
2. **路由**：从返回的中断负载取 `directed_to.user_id`，只推给该玩家；其余玩家收一条"等待 X 行动"的旁观消息即可。
3. **叙述推流**：`narrate` 节点继续用 `astream_agent_collect` 把 DM 叙述 token 流给**全房间**。
4. **重连**：因为状态在 checkpointer，玩家断线重连后，用同一 `thread_id` 重新取当前中断负载即可恢复"该你掷骰"的界面。
5. **超时兜底**（可选）：某玩家长时间不报骰，可设超时由引擎代掷（`Command(resume=引擎随机)`），避免卡死全房间。

## 6. 一个完整回合的中断时序（玩家攻击为例）

```
next_turn        → 选中玩家A（先攻第1位）
declare_action   → interrupt(declare_action) ──► 推给A："攻击/技能/道具/创意？目标？"
                 ◄─ resume {action_type:attack, attack_name:长剑, target_id:哥布林1}
resolve_action   → interrupt(attack_roll) ──► 推给A："掷 d20 + 5（也报伤害 1d8+3）"
                 ◄─ resume {d20:17, damage_result:7}
                 → 引擎：17+5=22 ≥ AC13 命中；哥布林1 HP 7→0 → 倒下
                 → 写 turn_events + combat_log
narrate          → DM 叙述（推流全房间）："长剑贯穿哥布林的咽喉…"
check_end        → 敌人未全倒 → continue → next_turn（轮到先攻第2位）
```
