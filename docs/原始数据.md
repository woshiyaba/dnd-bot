# 角色卡最小字段规范（v0·跑通用）

> 目标：用最少的字段，让引擎能跑通「探索 + 战斗」这条核心循环。
> 取舍原则：**引擎要读它、或循环要碰它的，才进卡**；纯属表演的留给 DM 临场生成，不进卡。
> 这一版刻意砍掉了 5e 里大量给人脑用的补偿规则（法术位、附赠动作、反应、借机攻击、完整状态列表、格子移动等），等核心循环验证好玩后再加。

---

## 0. 全局约定（引擎据此计算，不必逐个存进卡）

| 名称 | 规则 | 备注 |
|---|---|---|
| 核心判定 | `d20 + 属性调整值 (+熟练加值，若熟练)` 对比 `DC / AC`，达到或超过即成功 | 一切检定、攻击、豁免共用这一条 |
| 属性调整值 | `(属性值 − 10) ÷ 2`，向下取整 | 由属性值派生，**不单独存** |
| 熟练加值 | 1 级为 **+2**（本版先固定 +2） | 由等级派生，本版可写死 |
| 站位 | 用「区域」而非坐标（前排 / 后排 / 门口 / 祭坛旁…） | 只记角色在哪个区域 |
| 行动经济 | 每回合 **1 个主要行动**（+ 可选移动） | 不要附赠动作/反应 |

属性调整值速查（够用即可）：

| 属性值 | 8 | 10 | 12 | 14 | 16 | 18 | 20 |
|---|---|---|---|---|---|---|---|
| 调整值 | −1 | 0 | +1 | +2 | +3 | +4 | +5 |

## 1. 角色卡字段

「谁用」列：**引擎** = 确定性代码读取/计算；**DM** = 大模型用于叙述或判断；**两者** = 都要。

### 1.1 身份与外观（基本是给 DM 和玩家看的）

| 字段 | 类型 | 谁用 | 说明 |
|---|---|---|---|
| `id` | string | 引擎 | 唯一标识，引擎用它点名「轮到谁」 |
| `名字` | string | 两者 | |
| `种族` | enum | 两者 | 取自封闭花名册（如 人类/精灵/矮人/半身人…） |
| `职业` | enum | 两者 | 取自封闭花名册（如 战士/法师/盗贼/牧师…） |
| `等级` | int | 引擎 | 派生熟练加值、技能解锁 |
| `简介` | string | DM | 外貌/性格/动机，纯风味，可由大模型生成 |

### 1.2 六项属性（引擎判定的根，**只存属性值**）

| 字段 | 类型 | 谁用 | 说明 |
|---|---|---|---|
| `力量` | int | 引擎 | 近战、推拉、力量检定/豁免 |
| `敏捷` | int | 引擎 | 先攻、敏捷检定/豁免、远程/灵巧攻击 |
| `体质` | int | 引擎 | 影响 HP、体质豁免 |
| `智力` | int | 引擎 | 知识类检定/豁免 |
| `感知` | int | 引擎 | 察觉、洞察、感知豁免 |
| `魅力` | int | 引擎 | 社交、魅力豁免 |

> 调整值全部由这 6 个值现算，卡里不存。

### 1.3 战斗数值（引擎每回合都要读）

| 字段 | 类型 | 谁用 | 说明 |
|---|---|---|---|
| `当前HP` | int | 引擎 | 归零即「倒下/退出战斗」 |
| `最大HP` | int | 引擎 | |
| `AC` | int | 引擎 | 护甲等级，攻击检定的目标值。本版直接存最终值 |
| `先攻调整值` | int | 引擎 | 通常 = 敏捷调整值；战斗开始时 `d20 + 它` 定顺序 |
| `当前区域` | string | 引擎 | 角色此刻所在区域，决定谁够得着谁 |
| `存活状态` | enum | 引擎 | `正常 / 倒下`（本版先不做死亡豁免） |

### 1.4 熟练项（引擎据此决定检定/豁免要不要加熟练加值）

| 字段 | 类型 | 谁用 | 说明 |
|---|---|---|---|
| `熟练豁免` | list\<enum\> | 引擎 | 哪几项豁免加熟练加值（通常 2 项，由职业定） |
| `熟练技能` | list\<enum\> | 引擎 | 哪几项技能加熟练加值（如 隐匿/察觉/说服…） |

> 技能/豁免本身**不必逐项存数值**，引擎现算：`属性调整值 + (熟练 ? 熟练加值 : 0)`。

### 1.5 攻击手段（本版直接存算好的最终数值，最省事）

`攻击` 是一个列表，每个元素：

| 字段 | 类型 | 谁用 | 说明 |
|---|---|---|---|
| `名字` | string | 两者 | 如「长剑」「火焰弹」 |
| `命中加值` | int | 引擎 | 攻击检定时 `d20 + 它` 对比目标 AC |
| `伤害骰` | string | 引擎 | 骰子表达式，如 `1d8+3` |
| `伤害类型` | enum | 两者 | 挥砍/穿刺/火焰/毒…（影响抗性与叙述） |
| `射程` | enum | 引擎 | `近战 / 远程`，配合区域判断够不够得着 |

### 1.6 已学技能 / 资源（前面设计的技能系统接口）

`已学技能` 列表，每项引用「技能定义」并记录其消耗状态：

| 字段 | 类型 | 谁用 | 说明 |
|---|---|---|---|
| `技能id` | string | 引擎 | 指向封闭的技能定义（含效果积木） |
| `当前充能` | int | 引擎 | 如「每场战斗 1 次」剩几次 |
| `冷却剩余` | int | 引擎 | 若用冷却制，剩几回合可用 |

| 字段 | 类型 | 谁用 | 说明 |
|---|---|---|---|
| `资源池` | int | 引擎 | 可选：统一的法力/耐力。本版用「每战充能」即可不要它 |

### 1.7 背包（道具实例，引用封闭的道具定义）

`背包` 列表，每项：

| 字段 | 类型 | 谁用 | 说明 |
|---|---|---|---|
| `道具id` | string | 引擎 | 指向道具定义（其机械效果落在效果积木上） |
| `数量` | int | 引擎 | |

### 1.8 当前状态（一小撮就够，引擎每回合结算）

`状态` 列表，每项 `{ 状态: enum, 剩余回合: int }`。起步只要这几种：

| 状态 | 机械效果（落到效果积木） |
|---|---|
| `倒地` | 自身近战攻击劣势；攻击它的近战占优势 |
| `中毒` | 攻击检定与属性检定劣势 |
| `束缚` | 速度归零，无法移动出区域 |
| `眩晕` | 跳过自己的回合 |
| `持续伤害` | 每回合开始扣固定 HP（灼烧/流血） |

## 3. 怪物卡 = 角色卡的子集（让战斗能空转）

怪物不需要背包、升级、社交熟练这些。引擎跑战斗只需要：

```json
{
  "id": "mob_goblin_1",
  "名字": "哥布林",
  "当前HP": 7, "最大HP": 7,
  "AC": 13,
  "先攻调整值": 2,
  "当前区域": "前排",
  "存活状态": "正常",
  "攻击": [
    { "名字": "弯刀", "命中加值": 4, "伤害骰": "1d6+2", "伤害类型": "挥砍", "射程": "近战" }
  ],
  "状态": []
}
```

> 怪物的「这回合怎么打」由 DM（大模型）决定；命中、伤害、HP 全由引擎结算。

---

## 4. 这张卡足以跑通的循环

有了上面的字段，引擎已经能做完整一轮战斗：

1. 读所有参战者的 `先攻调整值` → 掷 `d20+它` → 排出行动顺序
2. 轮到某角色 → 读 `攻击` 或 `已学技能` → 玩家选，或 DM 替怪物选
3. 攻击检定：`d20 + 命中加值` 对比目标 `AC`
4. 命中 → 掷 `伤害骰` → 扣目标 `当前HP` → 必要时往目标 `状态` 加一条
5. 检定/豁免：`d20 + 属性调整值 (+熟练加值)` 对比 `DC`
6. 每个参战者 `当前HP ≤ 0` 即置 `存活状态=倒下`；一方全倒 → 结束
7. 结束后把 `当前HP / 背包 / 状态` 等写回世界状态库

需要继续的话，下一块自然是把**「效果积木库」「技能定义」「道具定义」**这三张表也定出来——它们就是上面 `技能id / 道具id / 状态` 这些引用真正指向的内容。
